# 无情台账机

> 专门做台账的AI工具：发票台账、合同台账...通通自动化！

## 背景

中小企业日常工作中存在大量重复性台账录入工作：
- 发票台账：发票号、金额、日期、供应商
- 合同台账：合同号、签约方、金额、期限

传统方式：人工手动录入 → 效率低、易出错

无情台账机：扫描/导入文档 → AI自动提取字段 → 生成结构化台账

**注意**：邮件台账因需要连接企业邮箱（涉及APP Secret Key申请），MVP阶段暂不支持。

## 核心功能

- [x] 文档信息提取（PDF/Word/图片）
- [x] 同义词字段映射（用户手动输入，不调用LLM）
- [x] 重复字段智能过滤
- [x] 结构化输出（Excel导出）
- [x] 置信度校验 + 人工确认

## 设计思路（核心亮点）

### 1. 用户共建同义词库

- 用户创建台账时定义基础字段（如"公司名"）
- 遇到新变体（如"企业名称"）时，手动添加到该字段的同义词列表
- **不调用LLM建议**，避免误识别（如把"company"误当字段名）
- 下次同类文件自动识别

### 2. 置信度校验 + 人工确认机制

- 批处理模式下，置信度低于80%的字段先自动填入
- 处理完成后，生成"待确认列表"
- 点击列表项 → 弹出原文件预览 + 高亮问题字段 → 人工核对/修改 → 确认
- **兜底机制**：确保极个别错误能被修正

### 3. MVP模板设计

- MVP提供3-5个常用模板（发票、合同）
- 语言作为模板属性（简中/繁中/英文）
- 用户可二次编辑模板字段
- **支持一张纸多张票据**：
  - 普通OCR无法自动拆分一页多票据（字段会混在一起）
  - 识别后**强制纳入人工复核**
  - 复核时核对字段信息 + 判断票据数量
  - 行数不足可插入新行，系统提醒手动完善字段
  - ⚠️ 这是**唯一需要全手工录入**的情况
  - **使用说明中会详细阐明处理方式**

### 4. 工作空间设计

- 每个用户拥有独立的工作空间
- 长期维护多个台账项目（发票台账、合同台账...）
- 新增条目自动延续序号（001→002→003...）
- **无需每次单独导出Excel**，台账数据持续累积
- 支持Excel导出备份

### 5. 数据保密

- 独立存储，不接入飞书或其他第三方SaaS
- 用户数据自主可控
- 企业版可私有化部署（规划中）

## 工作流程

```
用户：创建台账项目 → 定义基础字段
          ↓
导入文档（拖拽）→ AI提取字段 + 进度条显示
          ↓
置信度校验（阈值80%）→ 自动填入 → 低于阈值列入待确认列表
          ↓
用户：查看待确认列表 → 点击 → 原文件预览 + 高亮字段 → 确认/修改
          ↓
生成结构化台账 → Excel导出
```

## 技术栈

- 文档提取：阿里百炼文档智能 / 开源OCR（PaddleOCR）
- LLM：阿里百炼（Qwen系列）
- 存储：独立数据库（SQLite/PostgreSQL）
- 部署：Vercel / 私有化

## OCR选择说明

| 方案 | 优点 | 缺点 |
|------|------|------|
| 阿里百炼 | 效果好，支持PDF/图片 | 需要API Key |
| PaddleOCR | 开源可本地部署 | 效果略差，需要技术部署 |

**非中国大陆用户注意**：百度OCR需要内地身份证实名认证，建议使用阿里百炼或PaddleOCR。
