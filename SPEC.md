# 无情台账机 - 产品规格说明书 (SPEC)

## 1. 项目概述

### 产品定位
通用文档台账工具：不只是发票，任何可扫描的文档都能做台账！

### 核心价值
- **通用性**：不局限于某一类票据，支持用户自定义字段
- **自动化**：OCR 自动提取字段，减少人工录入
- **本地存储**：数据保存在本地 SQLite，不接入第三方 SaaS
- **在线 OCR**：调用云端 OCR API 进行文字识别

## 4. 功能规格

### 4.1 核心功能 (MVP)

| 功能 | 优先级 | 详细说明 |
|------|--------|----------|
| **文档导入** | 🔴 必须 | 支持拖拽导入，**必须支持 PDF 和图片**（JPG/PNG）<br>PDF 必须是核心格式，多页 PDF 自动拆分成单页处理 |
| **OCR 字段提取** | 🔴 必须 | 调用阿里百炼 API 识别文字并提取字段<br>支持图片和 PDF 两种格式 |
| **基础字段定义** | 🔴 必须 | 用户创建台账时可自定义字段（字段名、字段类型）<br>提供预设模板（发票、合同）|
| **同义词映射** | 🔴 必须 | 用户手动添加字段变体（如"公司名"↔"企业名称"）<br>不调用 LLM 建议，避免误识别 |
| **置信度校验** | 🔴 必须 | 80% 阈值，低于阈值进入待确认列表 |
| **待确认列表** | 🔴 必须 | 展示所有需要人工确认的条目<br>点击条目 → 弹出原文件预览 + 高亮问题字段<br>用户可修改后确认 |
| **Excel 导出** | 🔴 必须 | 导出当前台账的全部数据为 Excel 文件 |
| **数据存储** | 🔴 必须 | 本地 SQLite 持久化，支持多项目 |

### 4.2 文档导入详细要求

| 场景 | 处理方式 |
|------|----------|
| 单页 PDF | 直接 OCR 识别 |
| 多页 PDF | **自动拆分**，每页作为单独条目处理 |
| 单页图片 | 直接 OCR 识别 |
| 多张图片 | 每张图片作为单独条目处理 |
| 混合格式 | 按顺序处理，PDF 页面也算一个"图片" |

### 4.3 一页多票据检测

**需求**：检测一张纸上是否有多张票据（识别不精确没关系，检测到可疑就进人工复核）

**实现要求**：
- 技术方案：重复字段检测 + 坐标异常 + 布局分析
- 检测到可疑 → 强制标记为"待确认"，并在预览时提示"疑似多票据，请核对"
- **不需要精确识别每张票据的内容**，只需要标记"可能有异常"
- 工作量：1-2 天

### 4.4 工作空间

| 功能 | 说明 |
|------|------|
| 多项目支持 | 支持多个台账项目（发票、合同...） |
| 自动编号 | 条目自动延续序号（001→002...） |
| 数据累积 | 台账数据持续保存在本地 |

### 4.5 进阶功能 (规划中)

| 功能 | 说明 |
|------|------|
| 零样本提取 | 用户描述字段，大模型直接提取 |
| 少样本学习 | 提供 1-2 个示例，模型自动套用 |
| 多语言模板 | 简中/繁中/英文模板 |
| 私有化部署 | 企业版支持本地部署 |

## 5. 工作流程

```
用户：创建台账项目 → 定义基础字段
          ↓
导入文档（拖拽）→ PDF/图片 → OCR提取字段 + 进度条显示
          ↓
置信度校验（阈值80%）→ 自动填入 → 低于阈值列入待确认列表
          ↓
检测一页多票据（可疑 → 强制进入待确认）
          ↓
用户：查看待确认列表 → 点击 → 原文件预览 + 高亮字段 → 确认/修改
          ↓
生成结构化台账 → Excel导出
```

## 8. 注意事项

### 限制
- OCR 依赖网络连接
- 免费额度：每月 3000 页
- 并发限制：10 QPS
- 进阶版：引入大模型辅助提取（零样本/少样本学习）
- MVP 阶段暂不支持邮件台账（需企业邮箱授权）

### OCR 频率限制处理
为防止大量导入时卡住，需实现以下机制：
- **分批处理**：不会一次性全提交，排队逐个处理
- **队列机制**：配合 10 QPS 限制，排队慢慢处理
- **重试机制**：遇到限流自动等待后重试（指数退避）
- **进度条**：用户能看到实时处理进度

### 实名认证
- 阿里百炼支持境外护照/港澳通行证实名认证
- 客户需自行注册 API Key
